## 📋 المطلوب: تقرير منظم بالأقسام التالية

### 1. الهوية والغرض
- اسم المشروع (عربي + إنجليزي إن وُجد)
- الغرض من المشروع في 2-3 جمل
- نوع المشروع (اختر الأقرب أو وضّح):
  - موقع تعريفي / متجر إلكتروني / لوحة تحكم / ERP
  - منصة حجوزات / نظام إدارة محتوى / SaaS
  - نظام إنتاج وتصنيع / إدارة مشاريع / CRM
  - منصة فعاليات / موقع طبي / غير ذلك (وضّح)
- اللغات المدعومة + الاتجاه (RTL/LTR)
- الجمهور المستهدف
- **هل المشروع منشور؟** (الرابط إن وُجد + هل فيه مستخدمين حقيقيين؟)
- **النموذج التجاري:** (B2B / B2C / B2B2C / داخلي / Marketplace)

### 2. البنية التقنية
- Node.js version
- Backend Framework (Express / Fastify / Next.js / NestJS / ...)
- Frontend (React / Vue / Vanilla / EJS / Nuxt / ...)
- Database (PostgreSQL / SQLite / MongoDB / MySQL / ...)
- ORM (Drizzle / Prisma / Sequelize / TypeORM / None)
- Package manager (npm / pnpm / yarn)
- Authentication system (JWT / Sessions / Custom / OAuth)
- File storage (Local / S3 / Cloudinary / ...)
- Mobile App (إن وُجد - Expo / React Native / ...)
- المكتبات الرئيسية في package.json (top 10)
- محرك القوالب أو UI library (Tailwind / shadcn / Material / ...)

### 3. هيكل المشروع
- شجرة المجلدات الرئيسية (بدون node_modules)
- حجم المشروع (بدون node_modules)
- ربط GitHub:
  - مربوط؟ (نعم/لا)
  - اسم الـ repo
  - آخر commit (hash + message + date)
- ملف `.env`:
  - موجود؟
  - أسماء المتغيرات (بدون القيم)

### 4. قاعدة البيانات

- اسمه
- الأعمدة الرئيسية (مع أنواعها)
- العلاقات (Foreign Keys)
- عدد الصفوف الحالية
- مثال صف واحد (لو فيه بيانات)

### 6. الأدوار والصلاحيات
- ما الأدوار الموجودة حالياً؟
- صلاحيات كل دور بالتفصيل
- آلية المصادقة (Sessions / Tokens / JWT)
- بيانات الدخول الافتراضية لكل دور:
  ```
  الدور: ___ | البريد: ___ | كلمة السر: ___
  ```
- **هل كلمات السر مشفّرة؟ بأي خوارزمية؟** (bcrypt/SHA-256/plaintext)

### 10. نظام الفواتير والضريبة
- هل يوجد فواتير؟
- VAT calculation؟ (نسبة + الطريقة)
- ZATCA / QR Code؟
- الرقم الضريبي مدخل؟
- عدد الفواتير الحالية

### 11. مشاكل واجهة المستخدم
- مشاكل Responsive (Mobile/Tablet)
- مشاكل RTL/LTR (إن كان عربي)
- نصوص hardcoded في الكود
- TypeScript errors (شغّل `tsc --noEmit`)
- Console errors في المتصفح
- تنسيق غير متجانس
- خطوط مفقودة أو أيقونات مكسورة

### 12. الأمان (مهم جداً!)
- هل CORS مقيّد أم مفتوح؟
- هل فيه rate limiting؟
- هل الـ APIs محمية بـ middleware؟
- هل فيه endpoints حساسة بدون auth؟ **(اذكرها!)**
- هل Token ينتهي صلاحيته؟
- هل SESSION_SECRET قوي؟
- هل فيه CSRF protection؟
- هل فيه XSS protection؟
